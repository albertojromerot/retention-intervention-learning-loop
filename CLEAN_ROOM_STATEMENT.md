# Clean-Room Statement

This project is developed as a clean-room synthetic-data implementation.

The repository must not include:

- Real customer or member data.
- Employer or client data.
- Proprietary schemas.
- Confidential notebooks.
- Internal documentation.
- Screenshots from private systems.
- Organisation-specific metrics or thresholds.
- Copied code from private or restricted environments.

All business logic, data-generating assumptions, labels, model outputs, and documentation are synthetic and created for educational purposes.

The project is intended to demonstrate transferable machine learning, analytics engineering, intervention design, and governance practices in a generic financial-services context.
