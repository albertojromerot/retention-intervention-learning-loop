# Model Card — Synthetic Voluntary Attrition Intervention Learning Loop

## Project overview

This model card documents a clean-room, synthetic machine learning workflow for voluntary attrition prediction, intervention prioritisation, treatment logging, and post-intervention learning.

The project is designed as a portfolio-quality demonstration of responsible applied analytics engineering in a financial-services context.

It does not use, include, reproduce, reference, or depend on any real customer, member, employer, financial, operational, confidential, or proprietary organisation data.

## Intended use

The intended use of this project is educational and demonstrative.

It shows how an analytics team could build a governed retention-intervention learning loop using:

- Synthetic customer-month data.
- Feature engineering.
- A rules-based baseline.
- A first supervised machine learning model.
- An intervention-prioritisation policy.
- A synthetic treatment log.
- A treatment-aware second model.
- Governance checks.
- Model-comparison reporting.
- Portfolio visual reporting.

The project is not intended to make real customer decisions.

## Not intended for

This project should not be used for:

- Real credit decisions.
- Real customer treatment decisions.
- Real retention campaigns.
- Real eligibility, affordability, or risk decisions.
- Real financial-services production deployment.
- Automated decision-making affecting real individuals.

Any production use would require real data governance, legal review, model risk management, privacy assessment, fairness assessment, monitoring, and human oversight.

## Data

The project uses fully synthetic data generated by code in this repository.

The synthetic data represents a plausible financial-services customer-month panel with behavioural, engagement, relationship, and risk-like features.

The default synthetic profile contains:

- 25,000 synthetic customers.
- 24 monthly snapshots.
- 600,000 customer-month rows.
- A binary voluntary attrition target over a future 90-day window.
- Train, validation, and test splits.
- No missing values in the generated default pipeline output.

No real personal data is used.
